## 기록을 시작한 이유

새로운 기술을 배우는 것만큼 중요한 일은 **왜 사용했고, 어떤 문제를 해결했는지** 설명할 수 있는 상태로 만드는 것이라고 생각합니다. 이 블로그에는 단순한 사용법보다 문제를 발견하고 선택을 내린 과정을 중심으로 기록하려고 합니다.

## 다룰 내용

- Java와 Spring Boot 기반의 백엔드 개발
- PostgreSQL, Redis를 활용한 데이터 처리와 성능 개선
- Docker, Kubernetes, GitHub Actions를 이용한 배포와 운영
- 프로젝트에서 마주친 오류와 트러블슈팅 과정

## 기록 원칙

글은 다음 순서로 정리합니다.

1. 해결하려는 문제가 무엇이었는지
2. 원인을 어떻게 확인했는지
3. 어떤 선택지를 비교했는지
4. 적용한 방법과 결과는 무엇인지
5. 다시 한다면 무엇을 바꿀지

```java
public record LearningLog(
    String problem,
    String decision,
    String result
) {}
```

완벽한 정답보다 당시의 근거와 결과를 남기며 꾸준히 업데이트하겠습니다.
